# 🏛️ Architecture Notebook

> "A arquitetura de software não é sobre escolher tecnologias. É sobre tomar decisões fundamentadas para atender aos objetivos do negócio."

## Objetivo

O **Architecture Notebook** é um repositório de estudos, documentação e construção de conhecimento em Arquitetura de Software.

Seu propósito é consolidar conceitos, registrar decisões arquiteturais, documentar aprendizados e desenvolver o pensamento crítico necessário para projetar sistemas de software de forma consciente, escalável e sustentável.

Diferentemente de um conjunto de anotações tradicionais, este repositório busca organizar o conhecimento como uma base de referência pessoal, evoluindo continuamente conforme novos conceitos, padrões e experiências são adquiridos.

---

# Filosofia

Este repositório parte de alguns princípios fundamentais:

- Arquitetura é tomada de decisão.
- Toda decisão possui trade-offs.
- Não existem soluções universais.
- Tecnologias são ferramentas, não objetivos.
- A arquitetura deve servir ao negócio.
- O contexto sempre é mais importante do que a moda tecnológica.
- Todo conhecimento deve ser questionado, validado e evoluído continuamente.

---

# Objetivos de Aprendizagem

Este notebook tem como objetivos:

- Consolidar conceitos fundamentais de Arquitetura de Software.
- Desenvolver pensamento arquitetural.
- Entender os motivos que levaram ao surgimento de diferentes estilos arquiteturais.
- Registrar decisões utilizando ADR (Architecture Decision Records).
- Documentar padrões arquiteturais e padrões de projeto.
- Estudar atributos de qualidade e seus impactos nas decisões.
- Construir estudos de caso.
- Criar diagramas arquiteturais.
- Comparar diferentes abordagens para resolução de problemas.
- Registrar boas práticas e anti-patterns.
- Evoluir continuamente o conhecimento adquirido.
